Etapes du projet

  1. Créer la page de présentation des travaux
    • Récupérer les travaux depuis le back-end

  2. Créer la page de connexion
    • Gérer l'authentification de l'utilisateur
  3. Gérer les travaux
    • Supprimer des travaux via l'API
    • Ajouter des travaux avec les formulaires
    • Afficher dynamiquement les travaux
  4. Gérer les erreurs dans les formulaires

  • Gérer les événements utilisateurs
  • Manipuler les éléments du DOM
  • Récupérer les données utilisateurs via les formulaires

Description

Pour ce second projet, je découvre JavaScript pour la première fois en utilisant Node.js et le gestionnaire de paquets npm. A l'aide de la documentation Swagger, j'ai récupéré dynamiquement l'ensemble des travaux en utilisant la fonction fetch(). En utilisant des écouteurs d'événements au clic sur les filtres, je récupère les travaux ayant la même chaîne de caractère que celle de mon filtre. Pour l'authentification, j'utilise la méthode POST de fetch() pour envoyer le contenu du formulaire vers l'API. Si la connexion est autorisée, je stocke le token sur le localStorage. Après avoir construit la fenêtre modale, géré son ouverture et sa fermeture et récupéré les travaux, j'utilise la méthode DELETE de fetch() pour supprimer une photo ou toute la galerie d'un coup, que ce soit sur la modale ou sur la page d'accueil. L'objet FormData compile l'ensemble de paires clé/valeur du formulaire d'ajout avant de l'envoyer vers l'API avec la méthode POST de fetch(). Enfin, on va afficher dynamiquement la nouvelle image dans la modale en reconstituant entièrement tous les travaux (aucun rechargement de page).
