Back-end

Développer le back-end d'un site de
notation de livres

×

Etapes du projet

  1. Mettre en place de l'application
    • Créer un serveur Express simple
    • Créer une API RESTful
    • Mettre en place un système d'authentification
    • Ajouter une gestion des fichiers utilisateur
  2. Gérer la notation d'un livre
    • Gérer l'ajout d'une notation
    • Calculer la note moyenne

Compétences évaluées

  • Implémenter un modèle logique de données conformément à la réglementation
  • Mettre en oeuvre des opérations CRUD de manière sécurisée
  • Stocker des données de manière sécurisée

Description

Pour ce sixième projet, j'ai créé un serveur Express en utilisant le runtime NodeJS comme plateforme de développement. J'ai utilisé le package Mongoose pour configurer un schéma de données et structurer ma base de données NoSQL MongoDB. J'ai mis en place les opérations CRUD (Create, Read, Update, Delete) dans mes contrôleurs pour permettre la manipulation des données dans MongoDB. Pour gérer l'authentification et assurer la sécurité de ma base de données, j'ai intégré Bcrypt (algorythme de hachage du mot de passe), mongoose-unique-validator (assurer l'unicité des identifiants) et jsonwebtoken (générer et vérifier les jetons d'authentification) dans mes middlewares. J'ai ajouté une gestion des fichiers utilisateurs en utilisant Multer pour traiter les images entrantes et Sharp pour les adapter à l'application. J'ai intégré 2 fonctionnalités pour recueillir la notation des livres et calculer la note moyenne. Enfin, j'ai sécurisé mon application en configurant Cross Origin Ressource Sharing (CORS) et en utilisant des variables d'environnement pour protéger les informations sensibles.
